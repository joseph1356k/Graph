# Product Vision

## Idea central

Construir una interfaz extremadamente simple, al nivel de un bloc de notas, donde las personas puedan estructurar su conocimiento en Markdown y, a partir de ese conocimiento, orquestar sus agentes de Openclaw desde nuestro propio frontend.

## Tesis del producto

La nota no es solo un documento.
La nota es la interfaz principal para pensar, capturar intención y operar agentes.

## Qué queremos lograr

- que escribir sea la acción principal
- que la complejidad de agentes y automatización viva debajo de una interfaz mínima
- que el conocimiento capturado por el usuario se convierta en contexto operativo para Openclaw
- que el producto se sienta tan simple como un editor de texto, pero actúe como una consola de orquestación inteligente

## Qué NO queremos

- una UI cargada de paneles, dashboards y controles visibles todo el tiempo
- una experiencia estilo “tool for power users only”
- que el usuario tenga que aprender la arquitectura de agentes para usar el producto
- que el chat sea el centro visual de la experiencia

## Promesa del producto

El usuario escribe como si estuviera pensando en un bloc de notas.
El sistema interpreta ese conocimiento y lo vuelve acción orquestable.

## Resultado ideal

Una persona puede:

- escribir su visión
- bajar features
- capturar decisiones
- pensar en bloques de texto
- activar conversaciones contextuales justo donde está escribiendo
- y desde ahí controlar agentes de Openclaw sin cambiar de herramienta mental
