# UX Notepad Shell

## Principio principal

La interfaz debe sentirse igual de simple que el Bloc de notas de Windows.

## Estructura base

- editor de texto grande como superficie principal
- tabs simples
- acciones básicas: nuevo, abrir, guardar
- nada de navegación pesada
- nada de sidebars dominantes

## Filosofía visual

- cero ruido
- cero complejidad ornamental
- cero sensación de “suite enterprise”
- la escritura es el centro

## El chat dentro de esta visión

El chat no debe competir con el editor.
Debe aparecer como una capacidad contextual y secundaria.

## Regla de experiencia

Si una decisión de UI hace que el producto se sienta menos como escribir y más como “administrar software”, probablemente está mal.

## Objetivo

La persona debe sentir:

“estoy escribiendo una nota”

No:

“estoy operando un sistema complejo”.
